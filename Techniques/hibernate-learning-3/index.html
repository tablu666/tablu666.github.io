Hibernate学习（三）

今天主要学习Hibernate的多表操作。本菜鸡学习Hibernate的初衷是听说不用写SQL，那么就试试让Hibernate帮忙建表并且进行级联操作吧。

多表关系回顾

一对多

  • 从属关系，主表和从表。
  • 一对多建表，通过外键建立关系。
  • 员工管理表：一个部门有多名员工，一个员工只属于一个部门。

    多对多

  • 多对多建表，通过第三张表维护关系
  • 商品订单表：一个商品有多个订单，一个订单有多个商品。

删除

删除同样需要在配置Customer.xml中加入cascade=”delete”，用逗号隔开即可。


注意！如果没有配置直接调用delete方法会失败，原因是存在外键，实际上根据控制台打印的信息也可以看到Hibernate是先将外键置为null再进行两个表的删除操作。

修改

修改操作就是修改持久态对象的属性。

修改操作同样也需要配置，因为Hibernate双向维护外键，修改Customer时修改一次外键，修改Contact时也修改一次外键，造成效率问题。因此要让主表放弃外键维护，在Customer.xml修改inverse属性。
